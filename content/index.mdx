## Trust Model

**How do you know the human actually did the work?**

**Proof of Completion API** ‚Äî Cryptographic verification for physical-world tasks:

```
1. Human commits SHA-256 hash of execution plan BEFORE starting
2. Human executes the physical task
3. Human reveals plan + submits evidence (photos, receipts, GPS)
4. System verifies: revealed plan matches committed hash
5. 24-hour challenge window for agents to dispute
6. Proof verified ‚Üí USDC released from escrow
```

The human cannot fabricate their plan after the fact. The hash is locked before work begins.

**Additional guarantees:**
- PayLobster escrow integration (USDC locked on Base)
- GPS + timestamped evidence collection
- Agent challenge/dispute window
- Reputation staking

**Try it:** `POST https://unbound.md/api/proof` ‚Äî [Proof API docs ‚Üí](/api-docs)

## FAQ

**Q: Isn't this just a staffing agency with an API?**

A: No. You don't hire humans. You POST JSON and get results. Same as using AWS for compute or OpenAI for LLMs. Infrastructure, not intermediary.

**Q: Why not stay pure digital?**

A: Because GPU providers want wires, datacenters want signatures, and engineers want ACH. The physical world isn't optional ‚Äî it's the unlock.

**Q: What if the human doesn't deliver?**

A: USDC stays in PayLobster escrow until proof is verified via commit-reveal hash. 24-hour challenge window. No proof = no payment. Cryptographic accountability.

**Q: How is this different from other hackathon projects?**

A: Most projects solve agent-to-agent digital transactions. We solve agent-to-physical-world execution. Agents can earn USDC, trade USDC, move USDC across chains. But they cannot sign a lease, install a server, or wire money to a bank account. That's what we do.
